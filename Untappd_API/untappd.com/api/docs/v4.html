API Documentation (V4)

API Endpoints

The end point for all Untappd API is
http://api.untappd.com/v4

Client ID / Client Secret

In V4 of the API, we have added Client ID and Client Secrets. You can find them on your Dashboard page. You Client ID is your old API Key, while the Client Secret has been added. Whenever you are making a call to the API, you MUST pass both your Client ID and Client Secret as GET params like below:
http://api.untappd.com/v4/method_name?client_id=CLIENTID&client_secret=CLIENTSECRET

When making Authenticated requests, you will append the access token to the URL below.
http://api.untappd.com/v4/method_name?access_token=ACESSTOKENHERE See the Authentication section for more details.

Response

The responses in V4 has completely been re-written to match other APIs to make parsing easier. Responses are in the format below: 

"meta":{
      "code":200,
      "response_time":{
         "time": X.XXX,
         "measure":"seconds"
      }
   },
   "notifications":{
      "type":"notifications",
      "unread_count":{
         "comments":0,
         "toasts":0,
         "friends":0,
         "news":0
      }
   },
   "response":{  ... }}

In the header packet, we pass 2 special attributes that can help you track the usage of your API key:

								X-Ratelimit-Limit: XXXX
								X-Ratelimit-Remaining: XXX

In the event of an error, the format looks like this:

"meta":{
      "code":500,
      "error_detail":"The user has not authorized this application or the token is invalid.",
      "error_type":"invalid_auth",
      "response_time":{
         "time":0.002,
         "measure":"seconds"
      }
   },
   "response":[]

Additonal Information

In every beer attribtue, you will see the following values auth_rating and wish_list. These values refer to the authenticated user's rating and if they have added this beer to their wish list.

Note: By using the API, you state that your application is in accordance with our Terms of Use for our API.

Authentication

Authentication is handled via OAuth Authentication. You must edit your app and add in a redirect URL that Untappd will callback after the authorization has taken place. You will receive an accesss token to make an authenticated calls. At this time, tokens do not expire.

Server Side Authentication (Recommended)

NOTE: You must use HTTPS, and not HTTP when making authentication calls. All API level calls must be done with HTTP, however this authentication process must be sent over HTTPS

Direct the user to the following URL:
https://untappd.com/oauth/authenticate/?client_id=CLIENTID&response_type=code&redirect_url=REDIRECT_URL

After the user has authenticated and authorized the application, we will redirect back to the following address:
http://REDIRECT_URL?code=CODE

Take the code paramater and then pass it back to the following address via GET (pass parameters via the Query String). Ensure that the REDIRECT_URL you supply in this call, is the same that was supplied in the authenticate call:
https://untappd.com/oauth/authorize/?client_id=CLIENTID&client_secret=CLIENTSECRET&response_type=code&redirect_url=REDIRECT_URL&code=CODE

Finally, we will send back the access_token that will you use for future requests:
"meta":{ "http_code":200 }, "response" : { "access_token":{ "TOKENHERE" } }

Client Side Authentication

Direct the user to the following URL:
http://untappd.com/oauth/authenticate/?client_id=CLIENTID&response_type=token&redirect_url=REDIRECT_URL

After the user has logged in we will redirect back to the following address:
http://REDIRECT_URL#access_token=TOKENHERE

The Public Feed (Local)

Authentication: Not required

HTTP Method: GET

/v4/thepub/local

This method allows you the obtain all the public feed for Untappd, within a certain location. By default it will return at max 25 records.

Parameters

  • access_token or client_id and client_secret (required) - Either pass the access_token for authorized calls OR the client_id and client_secret for unauthorized calls
  • min_id (int, optional) - The numeric ID of the most recent check-in.
  • lng (float, optional) - The numeric Latitude to filter the public feed.
  • lat (float, optional) - The numeric Longitude to filter the public feed.
  • radius (int, optional) - The max radius you would like the check-ins to start within
  • max_id (int, optional) - The checkin ID that you want the results to start with
  • limit (int, optional) - The number of results to return, max of 50, default is 25

Foursquare Venue Lookup

Authentication: Not Required

HTTP Method: GET

/v4/venue/foursquare_lookup/VENUE_ID

This method will allow you to pass in a foursquare v2 ID and return a Untappd Venue ID to be used for /v4/venue/info or /v4/venue/checkins

Parameters

  • access_token or client_id and client_secret (required) - Either pass the access_token for authorized calls OR the client_id and client_secret for unauthorized calls
  • VENUE_ID (required) - The foursquare venue v2 ID that you wish to translate into a Untappd venue ID.
